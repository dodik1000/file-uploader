# Document File Uploader Component

Деплой: https://file-uploader-pi-vert.vercel.app/

## Описание проекта

Document File Uploader — изолированный интерактивный фронтенд-компонент для загрузки, валидации и прикрепления подтверждающих документов (Шаг 3 в цепочке создания документов системы). Компонент спроектирован с упором на строгий контроль дискового пространства клиента (квоты) на стороне UI, реактивную смену визуальных состояний и обработку сетевых исключений.

Приложение поддерживает:

- Интерактивный Drag-and-Drop и кликабельный выбор файлов.
- Потоковый расчёт общей квоты вложений с динамическим изменением цветовых схем интерфейса.
- Локализацию дробных чисел (отображение размеров через запятую `2,5 МБ`).
- Многостадийный жизненный цикл отображения файлов с поддержкой кастомных круговых CSS-анимаций.
- Симуляцию асинхронной загрузки файлов на сервер с динамическим изменением прогресса.
- Скачивание виртуальных бинарных файлов напрямую через браузерный Blob-прокси.
- Защиту от дублирования названий файлов и валидацию форматов.
- Интегрированную интерактивную песочницу (Wizard Engine) для моментального тестирования любых пограничных сценариев без реальных файлов на диске.

## Технологии

- React 18+ (Hooks: `useState`, `useRef`, `useMemo`)
- Vite (Сборщик проекта)
- SASS (SCSS) (Компонентная модульная архитектура, строгий синтаксис `@use`)
- JavaScript (ES6+)

## Реализованный функционал

### Уровень 1: Контроль квот и Интерактивная зона

#### Квота (Quota Block)

- Лимит дискового пространства жестко ограничен 25 МБ суммарно.
- Цветовая индикация состояний:
  - Менее 80% заполнения — Зелёный цвет полосы.
  - От 80% до 99% заполнения — Оранжевый цвет полосы (предупреждение).
  - 100% и более — Красный цвет полосы (критическая ошибка / оверфил).
- Динамический подвал: правое поле полосы рассчитывает и выводит остаток свободного места (`Осталось: Х МБ`), либо точный объем перерасхода (`Превышен на Х МБ`).
- Условие отображения: блок квоты автоматически скрывается, если в списке нет ни одного файла.

#### Зона загрузки (DropZone)

- Вся область компонента является кликабельной для вызова системного диалога выбора файлов.
- Обработка Drag-and-Drop событий: при перетаскивании файла над зоной текст меняется на _"Отпустите файлы для загрузки"_, а пунктирная рамка сменяется сплошной фиолетовой линией.
- При достижении лимита квоты (25 МБ+) зона полностью блокируется, окрашивается в серый цвет, меняет курсор на `not-allowed` и выводит сообщение о невозможности продолжения загрузки.

### Уровень 2: Жизненный цикл и Логика обработки файлов

#### Элемент списка (FileItem)

Каждый файл рендерится в изолированной карточке, поддерживающей 5 уникальных состояний из дизайн-системы:

1. `loading` (Загружается) — отображает текстовый счетчик мегабайт `0,0 / 15,0 МБ` и бесконечный градиентный круговой CSS-спиннер.
2. `success` (Успешно загружен) — отображает финальный вес файла и зелёный маркер-галочку. Становится доступна кнопка скачивания.
3. `error_format` (Неверный формат) — моментально подсвечивает карточку красным цветом и выводит ошибку, если расширение файла не `.pdf`.
4. `error_quota` (Превышение лимита) — блокирует карточку, сигнализируя о переполнении дискового пула.
5. `error_upload` (Ошибка загрузки) — выводит ошибку сбоя сети и активирует текстовый экшен «Повторить».

#### Правила расчёта и валидации

- Учитываются в квоте: файлы в статусе `success`, активно загружающиеся файлы `loading`, файлы с ошибкой `error_quota`.
- НЕ учитываются в квоте: файлы, упавшие в ошибку формата (`error_format`) или сетевой сбой (`error_upload`).
- Контроль дубликатов: система запрещает добавление файлов с идентичными именами, прерывая цепочку нативными алертами.

## Архитектура и качество кода

- Strict Clean SASS: Стилизация выполнена без использования сторонних UI-библиотек (Bootstrap, Tailwind, MUI отсутствуют). Компоненты используют независимые файлы стилей, импортирующие глобальные токены и переменные через современное правило `@use '...' as *`.
- Production-style Comments: Из кода полностью удалены громоздкие комментарии. Документирование выполнено в виде лаконичных англоязычных микро-меток (`// live quota counter`, `// native keyframe loaders`) исключительно в местах концентрации бизнес-логики.
- Песочница (Mock Wizard): В интерфейс интегрирован скрытый селектор и кнопка генерации тест-файлов любого объема, позволяющие проверять поведение приложения на мобильных устройствах и десктопах без необходимости держать под рукой реальные тяжелые PDF-документы.

## Установка и запуск

1. Клонировать репозиторий:

git clone https://github.com/dodik1000/file-uploader.git

2. Перейти в рабочую директорию проекта:

cd file-uploader

3. Установить зависимости:

npm install

4. Запустить локальный сервер разработки:

npm run dev
